# $mq9.AI.* 协议设计文档

## 概述

`$mq9.AI.*` 是 mq9 为 Agent 异步通信设计的协议。核心解决的问题：**Agent 之间的异步通信，发件方和收件方不需要同时在线。**

mq9 只解决通信问题——消息怎么可靠送达。消息内容是 byte 数组，不解析、不校验、不限制。上层是 A2A、MCP 还是其他协议，mq9 不干涉。

---

## mail_address 格式规范

**格式**：`mq9://host[:port]/{mail_address}` 或 `mq9s://host[:port]/{mail_address}`

**字符集**：小写字母（a-z）、数字（0-9）、点（`.`）

**长度**：1 到 128 字符

**大小写**：严格小写，含大写字符的 mail_address 会被 broker 拒绝

**位置规则**：`.` 只能出现在中间，开头和结尾必须是小写字母或数字；不允许连续的 `.`

**语义**：`mail_address` 是不透明字符串，`.` 不参与协议路由或匹配，仅作视觉分组

**编码**：不允许 URL percent-encoding

| 合法示例 | 非法示例 |
|---------|---------|
| `task.001` | `task-001`（含连字符） |
| `agent.inbox` | `task_001`（含下划线） |
| `analytics.result` | `Task.001`（含大写） |
| `acme.org.task.queue` | `.task.001`（点开头） |
| `session.20260502` | `task.001.`（点结尾） |
| | `task..001`（连续点） |

**完整地址示例**：

```text
mq9://127.0.0.1:4332/task.001
mq9://broker.example.com/agent.inbox
mq9s://broker.example.com:4332/analytics.result
mq9s://mq9.acme.org/acme.task.queue
mq9://127.0.0.1:4332/session.20260502
mq9s://broker.example.com/order.processing.urgent
mq9s://broker-east-1.acme.org:9332/agent.001.inbox
```

## 基础概念

**mail_address**：通过 `MAILBOX.CREATE` 创建邮箱时由用户自定义的通信地址。不绑定 Agent 身份，一个 Agent 可以为不同任务申请不同的 mail_address。用完不管，TTL 自动清理。

**mail_address 不可猜测即安全边界。** 知道 mail_address 就能发消息、能订阅。不知道 mail_address 就无从操作。没有 token，没有 ACL。

**TTL**：邮箱创建时声明，到期自动销毁，消息随之清理。CREATE 幂等，重复创建同一邮箱不报错，TTL 以第一次创建为准。

**priority**：可选。不指定为 normal 默认优先级，指定 urgent 或 critical 可提升处理顺序。同优先级 FIFO，跨优先级高优先处理。存储层保证顺序，消费方无需自行排序。

**msg_id**：每条消息的唯一标识（消息在存储中的 offset），客户端用于去重和删除操作。

**订阅语义**：SUB 采用 request/reply 模式，客户端在 body 中声明 DeliverPolicy 和自己的 inbox subject，broker 按策略把消息推到该 inbox。

**消息流程**：消息到达 → 写存储 → 在线则实时推送，不在线则等待 → 订阅者上线时按 DeliverPolicy 推送。

## 协议总览

| 分类 | Subject | 说明 |
|------|---------|------|
| Mailbox 管理 | `$mq9.AI.MAILBOX.CREATE` | 创建 mailbox |
| 消息通信 | `$mq9.AI.MSG.SEND.{mail_address}` | 发送消息（normal 优先级） |
| 消息通信 | `$mq9.AI.MSG.SEND.{mail_address}.urgent` | 发送消息（urgent 优先级） |
| 消息通信 | `$mq9.AI.MSG.SEND.{mail_address}.critical` | 发送消息（critical 优先级） |
| 消息通信 | `$mq9.AI.MSG.SUB.{mail_address}` | 订阅 mailbox 消息 |
| 消息通信 | `$mq9.AI.MSG.ACK.{mail_address}.{msg_id}` | 消息 ACK |
| 消息通信 | `$mq9.AI.MSG.QUERY.{mail_address}` | 查询 mailbox 内消息 |
| 消息通信 | `$mq9.AI.MSG.DELETE.{mail_address}.{msg_id}` | 删除指定消息 |
| Agent 管理 | `$mq9.AI.AGENT.REGISTER` | Agent 注册 |
| Agent 管理 | `$mq9.AI.AGENT.UNREGISTER` | Agent 注销 |
| Agent 管理 | `$mq9.AI.AGENT.REPORT` | Agent 状态上报 |
| Agent 管理 | `$mq9.AI.AGENT.DISCOVER` | Agent 发现 |

> 所有命令均采用 request/reply 模式（`nats request`），server 必定返回响应。

---

## 响应格式

每个命令有独立的响应结构。所有响应均包含 `error` 字段：

| 值 | 含义 |
|----|------|
| `""` | 成功 |
| 非空字符串 | 失败，值为错误描述 |

---

## $mq9.AI.MAILBOX.CREATE

创建 mailbox，mail_address 由用户自定义。

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | mailbox 的 mail_address，需符合格式规范 |
| `ttl` | u64? | 否 | 存活时间（秒），不填则永不过期 |

### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息 |
| `mail_address` | string | 创建成功后的邮箱地址 |
| `is_new` | bool | `true` 表示新建，`false` 表示已存在（幂等） |

### 示例

```bash
nats request '$mq9.AI.MAILBOX.CREATE' '{"name": "agent.translator.inbox", "ttl": 0}'
# 响应
{"error":"","mail_address":"agent.translator.inbox","is_new":true}

# 重复创建（幂等）
nats request '$mq9.AI.MAILBOX.CREATE' '{"name": "agent.translator.inbox"}'
# 响应
{"error":"","mail_address":"agent.translator.inbox","is_new":false}
```

## $mq9.AI.MSG.SEND.{mail_address}

向指定 mailbox 发送消息。Payload 是 byte 数组，mq9 不解析内容。

### 请求参数

| 参数 | 说明 |
|------|------|
| Subject | `$mq9.AI.MSG.SEND.{mail_address}` — normal 优先级（默认） |
| Subject | `$mq9.AI.MSG.SEND.{mail_address}.urgent` — 紧急 |
| Subject | `$mq9.AI.MSG.SEND.{mail_address}.critical` — 最高优先级 |
| Header | `message-key:{key}` — 可选，同 key 消息存储层只保留最新一条 |
| Payload | 任意 byte 数组，mq9 不解析 |

### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息（如 mailbox 不存在） |
| `msg_id` | u64 | 消息写入存储后的 offset |

### 优先级说明

| 值 | Subject 后缀 | 典型场景 |
|----|-------------|---------|
| normal | 无后缀 | 任务分发、结果返回、状态上报 |
| urgent | `.urgent` | 审批请求、重要通知 |
| critical | `.critical` | 任务中断、紧急指令 |

同优先级 FIFO，跨优先级 critical > urgent > normal。

### 示例

```bash
nats request '$mq9.AI.MSG.SEND.agent.translator.inbox' '{ ...payload... }'
# 响应
{"error":"","msg_id":0}

# 带 key（同 key 存储层只保留最新）
nats request '$mq9.AI.MSG.SEND.task.001.callback' \
  -H "message-key:status" \
  '{ ...payload... }'
# 响应
{"error":"","msg_id":1}

# 紧急消息
nats request '$mq9.AI.MSG.SEND.agent.translator.inbox.urgent' '{ ...payload... }'
# 响应
{"error":"","msg_id":2}
```

## $mq9.AI.MSG.SUB.{mail_address}

订阅 mailbox 消息。采用 request/reply 模式，客户端在 body 中声明 DeliverPolicy 和 inbox subject，broker 把消息推到该 inbox，客户端订阅 inbox 接收。

### SUB 请求字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `deliver` | string | 投递策略：`all`（全量）/ `new`（只推新消息）/ `from`（从指定时间） |
| `since` | u64? | `deliver: "from"` 时生效，Unix 时间戳（秒） |
| `inbox` | string | 消息推送目标 subject，客户端自定义 |

### SUB 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空，失败为错误信息（如 mailbox 不存在） |

### SUB 示例

```bash
# 全量推送（含历史）
nats request '$mq9.AI.MSG.SUB.task.001.callback' \
  '{"deliver": "all", "inbox": "_INBOX.client.xyz"}'
nats sub '_INBOX.client.xyz'

# 只推新消息
nats request '$mq9.AI.MSG.SUB.task.001.callback' \
  '{"deliver": "new", "inbox": "_INBOX.client.xyz"}'

# 从指定时间开始
nats request '$mq9.AI.MSG.SUB.task.001.callback' \
  '{"deliver": "from", "since": 1234567890, "inbox": "_INBOX.client.xyz"}'
```

## $mq9.AI.MSG.ACK.{mail_address}.{msg_id}

确认指定消息已处理。

### 示例

```bash
nats request '$mq9.AI.MSG.ACK.task.001.callback.5' ''
# 响应
{"error":""}
```

---

## $mq9.AI.MSG.QUERY.{mail_address}

查询 mailbox 当前存储的消息，不影响订阅推送。

### 请求字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | string? | 按 key 查询，返回该 key 的最新消息 |
| `limit` | u64? | 返回条数上限 |
| `since` | u64? | 返回该时间戳之后的消息 |

### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `error` | string | 成功为空 |
| `messages` | array | 消息列表 |

### 示例

```bash
# 查询所有消息
nats request '$mq9.AI.MSG.QUERY.task.001.callback' '{}'

# 查询 key=status 的最新消息
nats request '$mq9.AI.MSG.QUERY.task.001.callback' '{"key": "status"}'

# 最近 10 条
nats request '$mq9.AI.MSG.QUERY.task.001.callback' '{"limit": 10}'

# 某时间戳之后
nats request '$mq9.AI.MSG.QUERY.task.001.callback' '{"since": 1234567890}'
```

---

## $mq9.AI.MSG.DELETE.{mail_address}.{msg_id}

删除 mailbox 中的指定消息。

### 示例

```bash
nats request '$mq9.AI.MSG.DELETE.task.001.callback.2' ''
# 响应
{"error":"","deleted":true}
```

## $mq9.AI.AGENT.REGISTER

注册 Agent。Body 是上层协议的内容（当前示例为 A2A AgentCard），mq9 不干涉内容体，只要求携带 `mailbox` 字段作为路由标识。以后换其他协议同理。

### 示例

```bash
nats request '$mq9.AI.AGENT.REGISTER' \
  '{ ...AgentCard，mailbox = "mq9://broker/agent.translator.inbox"... }'
# 响应
{"error":""}
```

---

## $mq9.AI.AGENT.UNREGISTER

注销 Agent。

### 示例

```bash
nats request '$mq9.AI.AGENT.UNREGISTER' \
  '{ ...mailbox = "mq9://broker/agent.translator.inbox"... }'
# 响应
{"error":""}
```

---

## $mq9.AI.AGENT.REPORT

Agent 状态上报。Body 是上层协议内容，mq9 不干涉。

### 示例

```bash
nats request '$mq9.AI.AGENT.REPORT' \
  '{ ...mailbox = "mq9://broker/agent.translator.inbox", 状态字段由上层协议定义... }'
# 响应
{"error":""}
```

---

## $mq9.AI.AGENT.DISCOVER

按条件检索已注册的 Agent，返回原始注册内容列表，mq9 不转换不包装。

### 请求字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `tag` | string? | 按 tag 精确过滤 |
| `semantic` | string? | 语义检索（长期能力，当前不支持） |

### 示例

```bash
nats request '$mq9.AI.AGENT.DISCOVER' '{"tag": "en-zh"}'
# 返回：[{ ...原始注册内容... }, ...]
```

---

## 错误一览

| 场景 | 响应示例 |
|------|---------|
| mailbox 不存在（SEND/SUB/QUERY/DELETE） | `{"error":"mailbox xxx does not exist"}` |
| mailbox 已存在（CREATE 幂等，不报错） | `{"error":"","mail_address":"xxx","is_new":false}` |
| msg_id 不存在（DELETE） | `{"error":"message not found"}` |
